# UI Redesign Plan

## Goal

Make the site instantly clear.

When someone opens the homepage, they should understand in a few seconds:

- who Ricardo Gao is
- what Ricardo builds
- what kind of roles Ricardo fits
- where to find projects
- how the AI helper is useful

The site should feel like a focused personal portfolio, not a generic template.

## Current Problems

- too much template-like layout and filler structure
- weak first impression
- unclear value proposition above the fold
- top bar text like `Live on localhost` adds noise instead of value
- homepage reads more like a styled resume dump than a sharp portfolio landing page
- floating AI widget looks decorative instead of obviously useful
- login is over-emphasized for a portfolio site

## New Direction

The homepage should communicate this immediately:

`Ricardo Gao`
`AI + Data + Automation Builder`
`I build practical AI systems, RAG workflows, automations, and analytics products.`

Instead of generic dashboard language, the UI should feel:

- clear
- personal
- product-minded
- technical
- modern but restrained

## What Will Change

### 1. Rewrite the hero section

Replace the current hero copy with a sharper portfolio message.

The first screen should include:

- Ricardo name
- one-line role statement
- one short value proposition
- one small availability line
- 2 clear calls to action

Example structure:

- `Ricardo Gao`
- `AI Developer | Data | Automation | RAG Systems`
- `I build source-grounded AI tools, analytics workflows, and automation systems that turn messy data into useful products.`
- `Available for May 2026 co-op`
- buttons:
  - `View Projects`
  - `Ask Ricardo AI`

### 2. Remove noisy template elements

Remove or replace items like:

- `Live on localhost`
- `Agent Workspace`
- generic dashboard wording
- unnecessary chrome that does not help the viewer understand Ricardo

The top area should feel like a portfolio header, not a dev environment.

### 3. Reframe the homepage sections

The homepage should be organized by viewer intent:

1. who Ricardo is
2. what Ricardo builds
3. selected projects
4. why Ricardo is a fit
5. quick contact / resume actions

Instead of long resume-style blocks, convert content into clearer portfolio sections.

### 4. Make projects the real center of the page

Projects should be visually stronger than they are now.

Each featured project card should show:

- project name
- one-line problem solved
- stack
- impact / result
- optional CTA later like `View case study`

The goal is for recruiters to immediately see:

- Ricardo ships real things
- Ricardo works across AI, automation, and data
- Ricardo can connect technical work to business value

### 5. Reposition the AI widget

The floating AI widget should clearly explain its purpose.

It should feel like:

`Ask Ricardo AI`
`Paste a job description and I will tell you why Ricardo fits`

Suggested starter prompts:

- `Why is Ricardo a fit for this AI internship?`
- `Summarize Ricardo for a recruiter`
- `Compare Ricardo's skills to this JD`
- `What projects match an AI/data role?`

This makes the widget useful immediately instead of decorative.

### 6. Reduce emphasis on login

This is a personal portfolio first.

So login should not dominate the experience.

Changes:

- keep login available but visually secondary
- move it into a small top-right utility action
- do not let login compete with hero, projects, or personal brand

### 7. Tighten the writing

All copy should become shorter and more direct.

Rules:

- no vague startup-sounding filler
- no generic “builder console” language
- no wording that sounds like a template
- every section should answer a viewer question quickly

## Proposed Homepage Structure

### Above the fold

- Ricardo Gao
- AI Developer / Data / Automation
- one sentence value proposition
- availability
- two CTA buttons

### Selected Work

- 3 or 4 strong featured projects
- each with impact and stack

### Why Me

- short strengths section
- practical AI
- business intelligence
- automation
- fast product execution

### Experience Snapshot

- compact version, not a full resume wall

### Contact

- email
- LinkedIn
- GitHub
- resume download later if needed

## Visual Changes

- reduce dashboard feel
- make it look like a polished portfolio landing page
- cleaner header
- less sidebar heaviness
- stronger visual focus on hero and project cards
- better hierarchy and less text density
- AI widget should feel intentional and useful

## Implementation Order

1. simplify header and remove noisy text
2. rewrite hero section
3. rebuild homepage information architecture
4. redesign featured project cards
5. rewrite AI widget copy and prompts
6. reduce login visual priority
7. polish spacing, typography, and CTA hierarchy

## End Result

After the redesign, someone landing on the site should immediately think:

- this is Ricardo's portfolio
- Ricardo builds AI, data, and automation products
- Ricardo has real project depth
- Ricardo is relevant for technical co-op or internship roles
- the AI widget is useful for evaluating fit
